En HTML existen 2 tipos de etiquetas, de linea y de bloque, pero además existen otros 2 tipos según lo que contienen:
- Etiquetas que contienen solo texto
- Etiquetas que contienen otras etiquetas
Vamos a centrarnos en el segundo grupo que son las etiquetas que se utilizan para agrupar y asociar información relacionada pero que no tienen un significado semántico.
¿Para que sirve agrupar?
- Para identificarlo como una sección específica, como section aunque este si aporta un valor semántico.
- También puede ser necesario agrupar para darle un estilo visual (CSS), si quieres que un conjunto de información se vea diferente.
- También para posteriormente darle funcionalidad con JavaScript.
Debemos ser capaces de encontrar el equilibrio entre ser ordenados y tener etiquetas de más. Aunque ante la duda, mejor que sobren, luego será más fácil darle estilo.

Etiquetas vistas anteriormente

section, main, header, footer, article... Son también etiquetas de agrupación, no aportaban nada a nivel visual pero estructuran muchisimo semánticamente la web.

<div></div>

Es una de las etiquetas más utilizadas aunque es de las etiquetas que "menos sirven", no tiene ninguna funcionalidad mas que hacer grupos de información. Ni aporta valor semántico ni el navegador la interpreta de ninguna manera visual. Tiene todos los atributos comunes y de eventos. Puede contener otras etiquetas e incluso texto. 

                <div class="noticia">
                    <h3>Titulo noticia 4</h3>
                    <p>Descripción noticia</p>
                </div>

Esto puede ser un article en vez de un div. Las dos cosas son válidas pero semánticamente es más correcto un article. Un div puede tener varios div dentro. El ejemplo anterior podría tener un div padre "noticias-container" y dentro 3 o 4 noticias, que el conjunto poddría ser un section. Debemos tener una lógica entre agrupación y tener un orden semántico.
Abusar de los div puede causar una mala accesibilidad y funcione peor los SEO, el código ocuparía mucho más tamaño y tardar en cargarse y renderizarse.

<span></span>

Etiqueta muy conocida pero sirve de muy poco y se abusa mucho de ella. Tiene atributos comunes y de eventos. Sería como div pero de linea. Selecciona el texto para luego poder darle estilo con css o interactividad con JS. No aporta ninguna semántica ni cambia el flujo natural del texto.

<p>Este <strong> párrafo </strong> de texto tiene <span> algunos fragmentos de span.</span> </p>

Es una etiqueta de linea y puede contener más etiquetas de linea.

<blockquote></blockquote>

Ya vista, la repasaremos.
Se utiliza para indicar que el texto pertenece a otro sitio externo del cual se indica la fuente. Es una etiqueta de bloque y contiene atributos comunes y de eventos.
Tiene un atributo específico cite de tipo URL para indicar el origen de la cita.

<hr/>

Inserta una linea horizontal y se utiliza para separar semánticamente bloques de contenido de otros bloques de contenido. Es un elemento de bloque. Cada navegador la pintará a su manera hasta que no le digamos lo contrario, se puede personalizar.

<Listas ul, ol y dl>

ul y ol Son elementos de bloque y se utilizan para organizar información en formato de listado. Disponen de etiquetas hijas li. Ambas funcionan de la misma manera, lo único que cambia es el orden. En una van a aparecer viñetas (unorderer list) y en otra números (orderer list).

UL (Unorderer list), se utiliza para listar una serie de elementos en los que no importa el orden.
OL (Orderer list), se utiliza para listar una serie de elementos en los que importa el orden.
LI (List item), Elemento de bloque. Es cada uno de los elementos que contiene la lista. Dependerá de su padre (ul u ol) que aparezca con una viñeta o un número.
DL (Description list), se utiliza para listar elementos y definirlos. Usada en diccionarios p.ej. No dispone de etiquetas hijas li, tiene las suyas propias.
- DT -- Término a definir
- DD -- Definición del término

Lista desordenada

                <ul>
                    <li>Aceite</li>
                    <li>Huevos</li>
                    <li>Patatas</li>
                    <li>Sal</li>
                </ul>

Lista ordenada

                <ol>
                    <li>Freir patatas</li>
                    <li>Mezclar con los huevos</li>
                    <li>Freir todo</li>
                    <li>Comer la tortilla</li>
                </ol>

  3. Freir todo

                    <dt>Patatas</dt>
                        <dd>Que estén buenas y sean del año</dd>

Listas anidadas

Una lista puede tener listas anidadas, siempre dentro de un li. Debemos tener especial cuidado a la hora de abrir y cerrar las etiquetas.
Se pueden mezclar listas desordenadas con ordenadas y viceversa. 

                <ul>
                    <li>Ejemplo 1</li>
                    <li>Ejemplo 2
                        <ul>
                            <li>Ejemplo 2.1</li>
                            <li>Ejemplo 2.2</li>
                        </ul>
                    </li>
                    <li>Ejemplo 3</li>
                    <li>Ejemplo 4
                        <ol>
                            <li>Ejemplo 4.1</li>
                            <li>Ejemplo 4.2</li>
                            <li>Ejemplo 4.3</li>
                        </ol>
                    </li>
                    <li>Ejemplo 5</li>
                </ul>

Figure y figcaption

Es una etiqueta de agrupación para imágenes o elementos media como un vídeo. Normalmente lo vamos a encontrar con ina etiqueta img en su interior. Va a servir para que tenga una semántica mucho mayor que solo una img por sí sola y a nivel estructural nos va a ayudar a manejarlo luego con CSS.
Dentro de la etiqueta figure vamos a tener 2 etiquetas:
- La etiqueta img que va a contener la imagen.
- La etiqueta figcaption que va a tener la descripción de dicha imagen (que es como el alt de img) pero a nivel semántico tiene mayor valor , y a nivel visual el navegador lo va a renderizar como si fuese el pie de página de la imagen. 

                <figure>
                    <img src="./img/imagen_ejemplo.jpg" alt="Escudo casco">
                    <figcaption>Escudo Club Deportivo Casco Viejo</figcaption>
                </figure>
