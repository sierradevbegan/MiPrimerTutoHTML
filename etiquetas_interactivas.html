Etiquetas interactivas

Serán un conjunto de etiquetas que sirven para que el usuario pueda interactuar con nuestro contenido. Con ellas se podrán desencadenar algunas acciones en la web y vienen a suplir algunas características que antes solo podíamos hacer con JS.
El ejemplo más claro de etiquetas interactivas pueden ser las de formulario (select, checkbox...).
Debemos tener cuidado y no usarlas sin sentido. Antes de usar alguna etiqueta nueva debemos comprobar su compatibilidad para asegurarnos que tiene el mayor soporte posible. Podemos usar Can I Use. Ahí le podemos introducir una etiqueta o una propiedad CSS y ver que nivel de soporte tiene por cada uno de los navegadores.

Ocultación semántica

En muchas ocasiones, cuando queremos ocultar un elemento lo hacemos con CSS utilizando las propiedades display:none, visibility:hidden, opacity:0 o alguna similar.
Lo ideal sería tener una forma de saber si semánticamente tiene sentido que un elemento está oculto, por ejemplo, porque estamos enfocados en otra sección y la sección oculta no es relevante en este momento.
Con el atributo hidden hacemos desaparecer visualmente un elemento HTML (y todo su contenido, si está en una etiqueta contenedora).
Hasta ahora, para realizar esta tarea teníamos que recurrir a CSS, ya que se trata de una cuestión de presentación visual.

Código: 

                        <div>
                            <h2>Sección 1</h2>
                            <p>
                                Lorem ipsum, dolor sit amet consectetur adipisicing elit. Harum maiores debitis qui, aut architecto corrupti quis cupiditate labore dolores 
                                nemo dolorem facilis vel similique aspernatur, rem magnam a. Placeat, architecto.
                            </p>
                        </div>
                        <div hidden> <!-- Esta sección permanecerá oculta -->
                            <h2>Sección 2</h2>
                            <p>
                                Lorem ipsum dolor sit amet consectetur adipisicing elit. Inventore sapiente corrupti neque ut molestias laudantium modi nulla fuga excepturi 
                                magnam perferendis distinctio mollitia suscipit, sunt iusto rem, labore saepe quo?
                            </p>
                        </div>

Sección 1

Lorem ipsum, dolor sit amet consectetur adipisicing elit. Harum maiores debitis qui, aut architecto corrupti quis cupiditate labore dolores nemo dolorem facilis vel similique aspernatur, rem magnam a. Placeat, architecto.

Sección 2

Lorem ipsum dolor sit amet consectetur adipisicing elit. Inventore sapiente corrupti neque ut molestias laudantium modi nulla fuga excepturi magnam perferendis distinctio mollitia suscipit, sunt iusto rem, labore saepe quo?

El atributo hidden, no tiene nada extraño. Simplemente se trata de un atributo, que cuando un elemento lo incorpore, el mismo tendrá un display:none por defecto. Sin embargo, nosotros podemos cambiar el estilo mediante CSS si lo deseamos.

Etiquetas details y summary

Details, se trata de un elemento que muestra un texto y forma o icono identificativo que el usuario puede pulsar y desplegar (modo acordeón) una información que por defecto está contraida.
Se trata de una etiqueta contenedora que actúa como un div, salvo que todo el contenido de esta etiqueta no se muestra inicialmente.
Con summary podremos personalizar el texto que sale por defecto a la hora de usar una etiqueta details.

Código:

                        <details>
                            <summary>Ejemplo de summary</summary>
                            <section>
                                <p>
                                    Lorem ipsum dolor sit amet consectetur 
                                    adipisicing elit. Inventore sapiente 
                                    corrupti neque ut molestias laudantium 
                                    modi nulla fuga excepturi 
                                    magnam perferendis distinctio mollitia 
                                    suscipit, sunt iusto rem, labore saepe quo?
                                </p>
                            </section>
                        </details>

Ejemplo de summary

Lorem ipsum dolor sit amet consectetur
adipisicing elit. Inventore sapiente
corrupti neque ut molestias laudantium
modi nulla fuga excepturi
magnam perferendis distinctio mollitia
suscipit, sunt iusto rem, labore saepe quo?

*Con el atributo open (<details open>), el details por defecto va a estar abierto.

Dialog y popover

En HTML5 es posible crear ventanas de diálogo personalizadas sin necesidad de JS (o mediante muy poco JS), de forma nativa, e incluso crear ventanas modales. Todo ello se realizará mediante la etiqueta HTML dialog en diferentes modalidades:

- Ventana de diálogo: Se denomina así a una ventana que comunica el sistema o máquina con el usuario, para anunciarle información, pedirle confirmación, pedirle datos...
- Ventana de diálogo modal: Se denomina así a una variación de la anterior, donde se obliga al usuario a atenderla y responderla, impidiendo hacer ninguna otra tarea mientras.
