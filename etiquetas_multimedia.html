Etiquetas de contenido incrustado

Es coger un trozo de otra web e incrustarlo en nuestra web. Puede ser externa o interna.
- Normalmente se usarán para cargar contenido externo.
- También se puede cargar contenido de nuestra misma web.
- Es una referencia a un recurso almacenado en otra ubicación.
- Al estar enlazado, garantiza que si el original cambia, nuestra web mostrará la versión actualizada.

Podemos incrustar contenido como objetos multimedia, bloques de código HTML con contenidos diseñados por terceros e incluso webs enteras.
Para ello usaremos 2 tipos de etiquetas:
- embed
- iframe (casi siempre se utiliza esta)

Etiqueta embed

Podemos incrustar casi cualquier tipo de recurso:
- Documentos HTML (mejor usar iframe)
- Imágenes (mejor usar img)
- Videos (mejor usar video)
- Audio (mejor usar audio)
- Flash (ya no se utiliza)
- Controles ActiveX

Atributos de la etiqueta embed:
- src: La URL (interna o externa) del recurso que será incrustado y reproducido o ejecutado por el plug-in.
- type: Tipo de contenido (también conocido como "tipo de medio en internet" o "tipo MIME").
- width y height: Determinan el tamaño del contenedor en el que se va a incrustar el contenido (recomendado para informar al navegador como renderizar el espacio que debe reservar).

Ejemplo:

                        <embed type="video/webm" src="trailers/estreno.mov" width="640" height="480">

Existen etiquetas nuevas en HTML5 que son mucho mejores que usar embed.

Etiqueta iframe

Etiqueta de referencia para poder incrustar contenido externo (o interno). Representa un contexto de navegación anidado (una web dentro de otra web). Es una ventana que permite acceder a un recurso que se encuentra ubicado en un contexto distinto a la página que se está mostrando en el navegador.
Básicamente, permite incrustar un documento o parte de un documento web dentro de una página HTML (interna o externa).

Atributos:
- src: La URL del recurso incrustado.
- srcdoc: Contiene el código HTML que mostrar dentro del elemento iframe. Si existe este atributo, se impone a la referencia proporcionada por el atributo sr. (no se utiliza).
- sandbox: Permite proporcionar restricciones al documento incrustado en el elemento iframe con objeto de mejorar la seguridad de la página (vamos a incrustar una página pero no va a poder hacer nada con la página padre).
- allow: Directivas referentes a permisos de acceso a determinados elementos del dispositivo físico, tales como el acelerómetro (detecta si hemos girado el móvil, lo tenemos vertical, lo estamos moviendo...), la cámara, la geolocalización... HTML5 tiene APIs para poder acceder a elementos de nuestro navegador y de nuestro dispositivo. Con esto hacemos que ese iframe tenga permisos o no para poder utilizar esas características.
- allowfullscreen: Si existe la posibilidad, permite que dentro del iframe podamos llamar a la propiedad fullscreen de nuestro navegador, permite al contenido incrustado ocupar la pantalla completa.
- width y height: Permiten indicar el ancho y alto en píxeles.
- loading: Con el valor lazy el contenido se carga cuando este se haga visible; con el valor eager (valor por defecto) el contenido se carga inmediatamente.

¿Que ocurre si queremos colocar contenido de una web externa?
Por ejemplo servicios como Youtube, Vimeo, SoundCloud... u otros sitios similares. Se trata de servicios que ofrecen contenido externo para incrustar en nuestra página web, pero lo que nos proporciona no son imágenes, video o audio en formatos que reconozca el navegador, sino un enlace con contenido HTML muy diverso, que puede incluir JS, CSS, imágenes, videos y muchos otros recursos.
Content-Security-Policy establece que va a haber sitios web que no van a permitir ser incrustados en otros sitios web.

Ejemplo de iframe a una web externa:

                        <iframe src="https://cdcascoviejo.net" frameborder="0" width="1024" height="768"></iframe>

Ejemplo de iframe interno:

                        <iframe src="ejemplo_formulario.html" frameborder="0" width="1024" height="768"></iframe>

Ejemplo de iframe de Youtube:
En este caso desde youtube, pulsando en compartir podemos copiar directamente el código del iframe.

                        <iframe width="560" height="315" src="https://www.youtube.com/embed/zw8hbzT7DcA?si=u7EDnY5BfTNI-S_C" 
                        title="YouTube video player" 
                        frameborder="0" 
                        allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" 
                        referrerpolicy="strict-origin-when-cross-origin" allowfullscreen></iframe>

El atributo srcdoc (poco utilizado)

Por otro lado el atributo srcdoc nos permite indicar directamente el código fuente del iframe en su valor, algo que puede ser interesante para crear contenido autogenerado a demanda. 

                    <iframe src="http://pagina.com/apartado/"
                        srcdoc="<p>¡Hola! ¡Soy el <strong>texto del iframe</strong>!</p>">
                    </iframe>

Etiqueta video

En HTML5 se introduce la posibilidad de mostrar videos directamente desde nuestro navegador. De hecho, si arrastramos un video a la ventana del navegador, comprobaremos que comienza a reproducirse en él. Esto ocurre porque los navegadores son capaces de mostrar imágenes, reproducir video y audio.

Atributos básicos:
- src: Archivo o URL del video a reproducir. Obligatoria si no tiene etiquetas dentro.
- width y height: Indica el ancho y alto del video. *Recomendable establecerlo.

Atributos de control:
- autoplay: Comienza a reproducir el video automáticamente. Ojo, está sujeto a normas del navegador.
- loop: Vuelve a iniciar el video cuando finaliza su reproducción (en bucle).
- muted: Establece que el video se reproduzca con sonido muteado (silenciado).
- controls: Muestra los controles de reproducción (por defecto no se muestran).
- playsinline: Reproduce el video en linea, es decir, en su area de reproducción.

Autoplay con grandes limitaciones. En los navegadores se han introducido numerosas limitaciones para que el autoplay funcione. En mobile ya no funciona y en los navegadores se va a requerir que haya habido alguna interacción del usuario.

Atributos de precarga:
- poster: Muestra una imagen a modo de presentación.
- preload: Antes de que empieze a reproducir el video ya tenga una pequeña precarga en la memoria para que cuando le das a play ya se pueda reproducir y no tenga que esperar a cargar la 1ª parte del video.

Formatos compatibles:
* Formatos modernos:
- MP4: Codec x264, DivX H264 - Alta calidad. El códec x264 es libre. (buen soporte, el más recomendado).
- WebM: Codecs VP8, VP9 - Alternativa libre a MP$ de Google (buen soporte)
* Formatos de nueva generación:
- AV1: Basado en códecs VP10, Daala y Thor. Compite con HEVC/H.265 (soporte irregular)
- HEVC: Codecs x265, DivX HEVC. Futura evolución de MP4 (soporte irregular).
* Formatos más antigüos o no orientados a web:
- OGV: Codec Theora. Alternativa libre a MP4 (sin soporte en Safari).
- MKV: Codecs Matroska. Buena compresión. Potente (alto consumo CPU)
- AVI: Codecs XviD, DivX 3/5. Menor compresión que MP4 (anticuado).

Uso de Source: La etiqueta video también puede contener la etiqueta source igual que con las imágenes.

                        <video width="640" height="480">
                            <source src="video.mp4" type="video/mp4" />
                            <source src="video.webm" type="video/webm" />
                            <source src="video.ogv" type="video/ogg" />
                            <img src="imagen.png" alt="Video no soportado" />
                            Su navegador no soporta contenido multimedia.
                        </video>

Ejemplo de video:

                        <video width="680" height="460" controls poster="img/EscudoCasco.jpg" src="assets/casco.mp4"></video>

Etiqueta audio

Funciona exactamente igual que la etiqueta video pero sin video. También puede contener etiquetas source.

Formatos:
- MP3: Codec MPEG Layer3. Buena calidad (buen soporte).
- AAC: Codec Advanced Audio Coding. Mejora el MP3. Usado como audio en MP4 (buen soporte).
- OGG: Codec Ogg Vorbis. Buena calidad, alternativa libre a MP3 (cuidado con Safari).
- Opus: Codec Opus. Buena calidad, alternativa libre a MP3 (cuidado con Safari).
- FLAC: Codec Flac Audio Lossless. Muy buena calidad, sin pérdidas. Tamaño superior (buen soporte).
- WAV: Codec Wave Sound. Formato de Microsoft (está soportado, muy pesado).

Etiqueta canvas

Se utilizará para que librerías gráficas usen este contenedor a modo de lienzo para poder dibujar (mediante JS) y montar gráficos. *Ver w3s

Etiqueta map y area

Se utilizan para realizar secciones de una imagen que pueden enlazar a diferentes secciones. Es decir, podemos pinchar en alguna zona de la imagen para que nos lleve a algún sitio mediante un enlace. *Ver w3s
