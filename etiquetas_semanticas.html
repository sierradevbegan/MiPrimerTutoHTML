¿Qué son las etiquetas semánticas de estructura?

Son etiquetas (la mayoria de bloque) para agrupar y dividir los contenidos de nuestra página web y darle un nivel semántico a la estructura, tanto para el usuario, como para el SEO (motores de búsqueda como google) y tmbien para facilitar la accesibilidad (que sea accesible para cualquier navegador o usuario incluso si padece algún tipo de discapacidad).

<article></article>

Es una de las etiquetas semánticas de sección más importantes de HTML5. Sirve para definir partes con la suficiente importancia como para cosiderarse una entidad destacable con información a su alrededor.
Por ejemplo una web de móviles o cualquier producto, cada uno de ellos será un article porque tiene un nombre, una imágen, una descripción, un precio... es decir, todo el producto en sí tiene significado por si mismo.

<nav></nav>

Etiqueta que va a agrupar los menús de navegación de nuestra página web. Dentro del nav podrá haber una sucesión de etiquetas (como enlaces...) y esta a su vez es posible verla dentro de otras etiquetas de sección como el header, el footer,...
Ayudará sobretodo a los navegadores sin interfaz gráfica a no perderse por la web. Es común ver más de un elemento nav dentro de una misma página.

<header></header>

Etiqueta de bloque que encerrará a la zona común de la parte superior de nuestra web. Normalmente estará el logotipo, el nombre de la web, el menú de navegación... que se repetirá en todos los documentos de la página. Es posible encontrar dentro de un section o un article una etiqueta header.

<footer></footer>

Etiqueta que encerrará la zona común de la parte inferior de nuestra web. Podrá estar compuesto por un menú inferior, páginas legales, copyright... y demás componentes que se repiten en la parte inferior de todos los documentos de nuestra web.

<section></section>

Nos permite agrupar una sección de nuestro documento web. Para identificar que puede ser un section piensa por ejemplo en algún elemento de tu web al que le puedas poner un título. Por ejemplo, cada etiqueta de este documento es un section, cada una de las partes de una página web, cada capítulo de un documento, cada una de las pestañas o tabs de un cuadro de diálogo, cada sección numerada de un documento...

<main></main>

Va a ser la etiqueta que va a englobar el contenido principal de nuestra web. Normalmente estará entre el header y el footer. Suele ser elemento hijo de body o div.

<aside></aside>

Es la contraposición del main, englobará el contenido que no es de relevancia para nuestra web, es secundario o no tiene relación directa con el tema principal del documento. Menús o sidebars, publicidad, enlaces relacionados...

¿Qué son las etiquetas semánticas de texto?

Son etiquetas que nos ayudan a dar relevancia a partes del texto, suelen ser etiquetas de linea y aportan una valor semántico a nuestros textos. Estas no tienen un significado gráfico ni visual, otra cosa es que los navegadores las renderizen de varias maneras (negrita, cursiva, ...), solo se centran en el significado semántico. Por supuesto estas etiquetas también mejoran la accesibilidad y el SEO.

Algunas de ellas como strong y em ya las hemos visto anteriormente Etiquetas de texto en HTML

<time></time>

Es una etiqueta de linea y la vamos a usar para marcar texto que haga referencia a una fecha o una hora. Será importante para la accesibilidad. Dispone de un atributo datetime en formato timestamp en formato string.
<p>Abierto de <time>10:00</time> a <time>21:00</time> todos los días.</p>
<p>Tengo una cita el <time datetime="2025-05-15 18:30">Casco Viejo</time></p>

<address></address>

Va a ser utilizada para encerrar las direcciones o las formas de contacto que mostraremos en nuestra web. Puede ser una URL, una dirección, un email... Aportará mucho a nivel de accesibilidad.

NOTA: Para más ampliacion puedes comprobar la documentación oficial en w3schools

Etiquetas no recomendadas

Son etiquetas que se usaban en versiones antigüas de HTML, aunque aún funcionan, que renderizaban el texto pero que no aportaban ningún valor semántico al texto, solo visual.

<b> - Negrita (no valor semántico)
<i> - Itálic (no valor semántico)
<small> - Reduce el tamaño de la fuente (no valor semántico)
<u> - Negrita (no valor semántico)

Ahora en HTML vamos a tratar de estructurar la información, más adelante ya daremos estilo a nuestras páginas.

Ejemplo código fuente con estructura semántica

                <!DOCTYPE html>
                <html lang="es">
                    <head>
                        <meta charset="UTF-8">
                        <meta name="viewport" content="width=device-width, initial-scale=1.0">
                        <title>Ejemplo HTML Semántico</title>
                    </head>
                    <body>

                        <header>
                            <h1>Título de la Página</h1>
                            <nav>
                                <a href="#">Inicio</a>
                                <a href="#">Sobre Nosotros</a>
                                <a href="#">Contacto</a>
                            </nav>
                        </header>

                        <main>
                            <article>
                                <h2>Artículo 1</h2>
                                <p>Contenido del artículo...</p>
                            </article>
                            <article>
                                <h2>Artículo 2</h2>
                                <p>Contenido del artículo...</p>
                            </article>
                        </main>

                        <aside>
                            <h2>Barra Lateral</h2>
                            <p>Información adicional...</p>
                        </aside>

                        <footer>
                            <p>Copyright © 2024</p>
                        </footer>

                    </body>
                </html>
