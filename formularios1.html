¿Qué es un formulario?

Se conoce como formularios a los mecanismos para enviar información por parte del usuario, a traves de unos campos visuales de forma sencilla e intuitiva, que determinan la naturaleza del tipo de información que se va a enviar y como hacerla llegar a otro extremo donde se procesará esa información.
El proceso de inserción será lo mas sencillo y cómodo posible. Trataremos de reducir los errores al introducir los datos y si no comunicar clara y rápidamente el error al usuario.

Form

Es una etiqueta de bloque con atributos propios. Englobará todo el contenido del formulario. Pueden coexistir varios formularios en una misma página.
La información que se puede enviar en un formulario pueden ser textos, nombres, emails, cantidades numéricas, teléfonos, archivos...

Atributos propios de form

action: Dirección URL del backend al cual se enviará la información obtenida en el formulario.

method: Método HTTP de envío. GET a través de URL y POST para envío extenso de información.

name: Nombre del formulario. Útil para identificar el formulario y procesar posteriormente.

target: Nombre del lugar donde se abrirá el formulario (_blank para pestaña nueva).

enctype: Codificación para el envío del formulario. Importante para el envío de archivos.

accept-charset: Fuerza a utilizar una codificación en los parámetros de texto del formulario.

autocomplete: Activa on o desactiva off el autocompletado para todos los campos del formulario.

novalidate: (Boolean) Descativa la validación HTML5 del formulario (desaconsejado).

Ejemplos de etiquetas que pueden encontrarse en un formulario

input type="text" Información de texto corto: Nombres, apellidos, textos cortos...

textarea Información de texto largo: Comentarios, observaciones...

input type="number" o input type="range" Números o cantidades: Edades, cuantías, precios...

input type"date" ó derivados Fechas u horas: Fecha de nacimiento, inicio de evento...

input type="checkbox" Verdadero/falso, Si/No, Opción A/B, On/Off...

input type="radio" o select Opción única: Elegir una opción de 2 o más posibles.

select multiple o input type="checkbox" Varias opciones: Elegir varias opciones de 2 o más posibles.

datalist Opción única abierta: Elegir una opción o indicar una propia.

input type="color" Selección de color: Escoger un color o tonalidad.

input type="file" Selección de archivo: Escoger un archivo para enviar.

GET

Recoge todos los datos de un formulario y los va a concatenar con la URL. Con GET podemos enviar como máximo 500 bytes de información.
No permite el envío de archivos adjuntos con el formulario.
Los datos enviados se visualizan en la barra de direcciones del navegador.
Despues del nombre de dominio aparecerá el símbolo ? y a continuación la clave/valor del primer campo que envía el formulario, el resto de parámetros clave valor se separarán entre sí mediante el símbolo &.
Determinados caracteres no son recibidos en la URL de la misma forma exacta en que fueron escritos, ya que tienen que ser codificados ya que en una URL no puede haber espacios, acentos, caracteres especiales....
Ejemplo: 

http://www.example.com:80/path/to/myfile.html?key1=value1&key2=value2#SomeWhereInTheDocument

Get se va a utilizar única y exclusivamente para formularios de consulta. No puede crear, modificar ni eliminar registros nuevos.

POST

Con POST vamos a mandar la información de manera oculta, es decir, no va a aparecer nada en la URL del navegador, no significa que vaya cifrada.
Se utiliza para enviar una gran cantidad de datos a la vez o cuando tenemos que enviar datos adjuntos, crear un registro, modificarlo o eliminar un registro. CRUD (Create, Read, Update, Delete).

INPUT

Es la etiqueta que más vamos a utilizar cuando estemos haciendo un formulario. Es el elemento principal donde podremos escribir la información que queremos mandar al servidor.
Dependiendo de la naturaleza de los datos usaremos input de un tipo o de otro.
Valores del atributo type: button, checkbox, color, date, datetime-locale, mail, file, hidden, image, month, number, password, radio, range, reset, search, submit, tel, text, time, url, week
