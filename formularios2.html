Resto de inputs

checkbox, color, date, datetime-local, image, month, radio, range, reset, search, tel, time, week.

Recordemos la importancia de los atributos name y value. Cuando mandamos los datos al servidor se van a enviar en parejas de 2. Estos son los campos que va a recibir el backend para poder ser tramitados. Es importante que el formulario se vea bien en HTML pero estar seguros de que cada input tenga el name correspondiente. El atributo name no debe contener caracteres especiales ni espacios... solo nombres en minúscula y si es necesario concatenarlos con un guión bajo.

Search

Es un input de texto pero que va a tener una funcionalidad extra, el navegador va a renderizar una x a la derecha del campo para burrar el contenido del input.

                        <input type="search" name="q" value="Buscar">

Image

No es mas que un input de tipo submit, es decir, un botón para enviar el formulario, pero que en lugar de un botón lo que va a renderizar es una imagen que le asociemos.

                        <input type="image" src="imagen.gif" alt="boton" weight="12" height="12">

Color

Se va a utilizar para seleccionar un color. Lo que va a salir es un colo-picker. No es más que un selector de color que renderiza el navegador o el SO. Se puede selecionar RGB, hexadecimal...

                        <input type="color" name="color">

Date

Con date podremos seleccionar una fecha del calendario que nos sale en ese input. Es un calendario nativo del SO. Valida que una fecha sea correcta.

                        <input type="date" name="fecha_nacimiento">

Time

Es un input que nos permite seleccionar una hora. Valida hora correcta

                        <input type="time" name="meeting_hour">

Datetime-local

Nos va a permitir introducir una fecha y una hora juntos. Valida que fecha y hora sean correctas.

                        <input type="datetime-local" name="fecha_hora">

Month

Nos permitirá seleccionar un més válido (de un año concreto).

                        <input type="month" name="mes">

Week

Selecciona un número de semana del año.

                        <input type="week" name="semana">

Tel

Nos permite almacenar un teléfono. Se le puede añadir un patron pattern="[0-9]{3}-[0-9]{2}-[0-9]{3}" por ejemplo, para que valide que es un número de teléfono correcto,. Pero claro esto es complicado porque cada país tiene un patrón diferente..

                        <input type="tel" name="teléfono" pattern="[0-9]{3}-[0-9]{2}-[0-9]{3}">

Range

Nos va a dar un deslizador para poder introducir un valor numérico. Lo normal es ponerle un mínimo y un máximo. Acepta el atributo step para desplazarse de 2 en 2 o en decimales .5 o .1.
Indique su experiencia en el parque de atracciones: 

                        <input type="range" name="experience" min="0" max="10">

Radio

Se enviará un único valor de entre las opciones disponibles. Son excluyentes. Por ejemplo el sexo o el color de unas zapatillas que compras. En los radiobutton es necerasio usar el value, ya que el name será el mismo para todas las opciones.

                        <input type="radio" name="color" value="negro"> Negro
                        <input type="radio" name="color" value="blanco"> Blanco

Hacerlas accesibles:

Lo normal es que cuando pinchamos en el texto de un checkbox o un radiobutton (o en el texto de cualquier otro input) se seleccione esa opción o campo correspondiente. Esto es ACCESIBILIDAD y se dá con la etiqueta label.

Etiqueta label

Se usa para asociar una descripción o texto explicativo con un elemento del formulario mediante el atributo for, que debe coincidir con el atributo id del control (input). Alternativamente puede anidar directamente el control dentro de la etiqueta label. 

                    <input type="radio" name="color" value="negro" id="black">
                        <label for="black">Negro azabache</label>
                    <input type="radio" name="color" value="blanco" id="white">
                        <label for="white">Blanco perla</label>


LABEL VS PLACEHOLDER VS TEXT SIMPLE
Es más correcto tener un label para cada input. P ej si anulamos el texto y solo ponemos un placeholder una vez rellenos puede que no sepamos que es cada campo y eso no es accesible. Aunque se ponga el placeholder, deberá llevar un texto en un label o como mínimo un texto simple.

Textarea

Crea una caja de texto me múltiples líneas donde los usuarios pueden escribir texto más extenso. Normalmente no tendrá value.
Atributos:
- rows Altura visible del texto en lineas
- cols Ancho visible del textaera en caracteres
- name, value, placeholder
- disabled Deshabilita el area de texto para que el usuario no modifique
- readonly Permite que el area sea visible pero no editable
- wrap Controla como se manejan las lineas de texto largas en el area de texto (hard, soft, off). 

                    <label for="biograph">Resume tu biegrafía</label>
                        <textarea id="biograph" name="bio" rows="4" cols="50"></textarea>


Select

Define una lista desplegable que permite al usuario elegir una opción de una lista de opciones. 

                    <label for="size">Seleccione la talla</label>
                    <select name="size" id="size">
                        <option value="">Elige</option>
                        <option value="38">Talla 38</option>
                        <option value="39">Talla 39</option>
                        <option value="40">Talla 40</option>
                        <option value="41">Talla 41</option>
                        <option value="42">Talla 42</option>
                    </select>


ATRIBUTO MULTIPLE
Existen desplegables que permiten seleccionar múltiples opciones. 

                    <label for="cars">Seleccione marca/s vehículo</label><br/>
                    <select name="cars" id="cars" multiple size="2">
                        <option value="volvo">Volvo</option>
                        <option value="saab">Saab</option>
                        <option value="opel">Opel</option>
                        <option value="audi">Audi</option>
                        <option value="bmw">BMW</option>
                    </select>


BOTTON VS SUBMIT

Los 2 sirven para enviar el formulario. Es preferible usar button, mas que nada porque los input de tipo submit tienen estilos heredados de los formularios que a veces suelen entrar en conflicto con nuestros estilos css mientras que button a la hora de darle estilo se comportará mejor.
